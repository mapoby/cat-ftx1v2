Insert this block after `uses: actions/checkout@v4` and before `# ── 2. Azure login`:

```yaml
      # ── 1b. Test gate (must pass before any build work) ──────────────────
      - name: Set up Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '22'
          cache: 'npm'

      - name: Install dependencies
        run: npm ci

      - name: Run tests
        run: npm test

      - name: Upload coverage
        uses: actions/upload-artifact@v4
        with:
          name: coverage-${{ github.sha }}
          path: coverage/
          if-no-files-found: warn
```

Notes:
- `actions/setup-node@v4` — current major version as of research date
- `actions/upload-artifact@v4` — current major version as of research date
- `node-version: '22'` — matches the node:22-alpine image used in the Dockerfile
- `cache: 'npm'` — caches node_modules keyed on package-lock.json; speeds up subsequent runs
- `npm ci` — uses the lockfile (package-lock.json must be committed — see INFRA-01 which is Phase 4; for now, npm ci requires the lockfile; commit package-lock.json as part of this plan)
- `npm test` — runs `vitest run` (non-interactive, exits non-zero on failure, halts the job)
- `if-no-files-found: warn` — does not fail the CI run if coverage/ is absent (e.g., tests skipped); only uploads when present

IMPORTANT: package-lock.json must be committed to the repository for `npm ci` to work. If it is not yet committed, run `npm install` locally first to generate it, then commit it. Check if it exists:

If absent, `npm ci` will fail in CI. The lockfile must be added to git in this plan.
