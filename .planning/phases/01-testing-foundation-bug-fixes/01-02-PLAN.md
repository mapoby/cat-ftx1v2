  <action>
Create vitest.config.mts in the project root (c:/GitHub/cat-ftx1v2/vitest.config.mts) with this exact content:

```typescript
import { defineConfig } from 'vitest/config'

export default defineConfig({
  test: {
    environment: 'node',
    include: ['test/unit/**/*.{test,spec}.ts'],
    coverage: {
      provider: 'v8',
      include: ['composables/catParser.ts'],
      reporter: ['text', 'lcov'],
    },
  },
})
```

Use the `.mts` extension (not `.ts` or `.js`) because package.json does not have `"type": "module"`. The `.mts` extension tells Node to parse this file as ESM regardless of the package type setting.

The `include` coverage array targets only `composables/catParser.ts` — this is the only file with testable pure logic in Phase 1.
  </action>
