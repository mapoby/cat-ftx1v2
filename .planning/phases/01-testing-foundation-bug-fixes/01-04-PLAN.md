Replace the entire `scanMemoryChannels` function body with the fixed version:

```typescript
export async function scanMemoryChannels(from = 1, to = 99): Promise<void> {
  const origFreqResp = await _sendAndWait('FA', 1500)
  const origFreq = origFreqResp.substring(2)

  try { await _sendAndWait('AI0', 1000) } catch { }

  await send('VM011')
  try {
    for (let i = from; i <= to; i++) {
      if (!state.value.connected) break
      await readMemoryChannel(i)
    }
  } finally {
    try {
      await send('VM000')
      await send('FA' + origFreq)
    } catch { }
    try { await _sendAndWait('AI1', 1000) } catch { }
  }
}
```

Changes:
1. Before the loop: `_sendAndWait('FA', 1500)` captures the current VFO frequency. `origFreqResp.substring(2)` strips the 'FA' prefix to get the raw 11-digit frequency string (e.g., '00014225000').
2. Before `VM011`: disable AI mode with `_sendAndWait('AI0', 1000)` — prevents unsolicited frames from corrupting solicited queue responses during bulk reads.
3. The `for` loop is wrapped in `try/finally`.
4. In `finally`: first restore VFO mode (`VM000`) and frequency (`FA` + origFreq), then re-enable AI mode (`AI1`). Both restores are in their own inner `try/catch` so an error in VM000 does not prevent AI1 from being sent.
