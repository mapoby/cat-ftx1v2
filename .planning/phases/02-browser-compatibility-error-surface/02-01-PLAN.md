<task type="auto">
  <name>Task 1: Add inline browser-gate script to nuxt.config.ts</name>
  <files>nuxt.config.ts</files>
  <action>
Add an `app.head.script` entry with `tagPriority: 'critical'` so Nuxt injects it before the module bundle.

The script must:
1. Check `'serial' in navigator` (hasSerial) and `window.isSecureContext` (isSecure).
2. If both are true — return immediately (Chrome/Edge on HTTPS; no gate rendered).
3. If hasSerial is true but isSecure is false — show the HTTPS-required message (per COMPAT-02).
4. Otherwise (no serial support at all) — show the wrong-browser message (per COMPAT-01).
5. Append the `div#compat-gate` to `document.body`. Guard with `readyState === 'loading'` to handle both cases: if DOMContentLoaded has not yet fired, attach a listener; if it already fired (script ran late), call immediately. Pattern: `if (document.readyState === 'loading') { document.addEventListener('DOMContentLoaded', showGate) } else { showGate() }`

Exact innerHTML for the gate div must contain:
- A heading: "FTX-1 CAT Controller"
- A styled message paragraph using the two distinct strings below
- HTTPS message (COMPAT-02): "This page must be served over HTTPS. Web Serial requires a secure context — plain HTTP is not supported."
- Wrong-browser message (COMPAT-01): "Web Serial API is not supported in this browser. Use Google Chrome or Microsoft Edge (desktop)."
- A `z-index: 9999` overlay so it covers any partially-rendered Vue content

Gate div style (inline cssText):
```
position:fixed;inset:0;display:flex;flex-direction:column;align-items:center;
justify-content:center;background:#0d1117;color:#e6edf3;
font-family:system-ui,sans-serif;padding:2rem;text-align:center;z-index:9999
```

The script must be a self-invoking function (IIFE) so it does not leak variables.

After adding the script, `nuxt.config.ts` still must not enable SSR — keep `ssr: false`.
