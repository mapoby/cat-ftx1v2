# Technology Stack

**Analysis Date:** 2026-05-08

## Languages

**Primary:**
- TypeScript 5.x — all application logic (`composables/`, `pages/`, `components/`)
- Vue 3 (SFC) — UI components (`.vue` files)

**Secondary:**
- Bicep — Azure infrastructure as code (`infra/main.bicep`)
- Bash — provisioning script (`infra/setup.sh`)

## Runtime

**Environment:**
- Node.js 22 (Alpine) — build stage only (`node:22-alpine` in `Dockerfile`)
- No Node.js runtime in production; output is static files served by nginx

**Package Manager:**
- npm (no lockfile committed — `package-lock.json` not present in repo root)

## Frameworks

**Core:**
- Nuxt 3 `^3.13.0` — SSR disabled (`ssr: false`), used purely as static site generator
  - Config: `nuxt.config.ts`
  - Build output: `.output/public/`
- Vue 3 (bundled with Nuxt) — reactive UI framework

**Build/Dev:**
- `nuxt generate` — static site generation, run inside Docker builder stage
- `nuxt dev` — local dev server (satisfies HTTPS/localhost requirement for Web Serial API)
- `vue-tsc ^2.0.0` — TypeScript type checking for Vue files
- TypeScript `^5.0.0` — compiler/type-checker

**Web Server (Production):**
- nginx `1.27-alpine` — serves static output, SPA fallback, gzip, asset caching
  - Config: `nginx.conf`

## Key Dependencies

**Production:**
- `nuxt ^3.13.0` — framework (includes Vite, Nitro build tooling, Vue 3)
- `magic-regexp ^0.11.0` — used in application code (regex builder utility)

**Development:**
- `@types/node ^20.0.0` — Node.js type definitions
- `typescript ^5.0.0` — TypeScript compiler
- `vue-tsc ^2.0.0` — Vue-aware TypeScript checking

## Configuration

**Build:**
- `nuxt.config.ts` — SSR off, devtools off, `APP_VERSION` and `buildDate` exposed as runtime config
- `tsconfig.json` — extends `.nuxt/tsconfig.json` (auto-generated by Nuxt)
- `APP_VERSION` build arg passed via `docker build --build-arg APP_VERSION=v1.<run_number>`

**Environment:**
- No `.env` required for local dev
- `APP_VERSION` env var consumed at build time via `nuxt.config.ts` `runtimeConfig.public`

## Platform Requirements

**Development:**
- Node.js 22
- Chrome or Edge (Web Serial API support required)
- HTTPS or `localhost` origin (Web Serial API browser constraint)

**Production:**
- Docker (multi-stage build)
- nginx 1.27-alpine container
- Azure App Service Linux container (B1 SKU minimum)
